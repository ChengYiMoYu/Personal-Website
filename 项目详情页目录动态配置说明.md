# 项目详情页目录动态配置说明

## 📖 目录是如何动态工作的

项目详情页的左侧目录是**完全动态**的，它会根据你在每个项目页面中定义的 `sections` 数组自动生成。

---

## 🎯 工作原理

### 1. 定义章节列表

在你的项目页面文件（如 `app/projects/personal_website/page.tsx`）顶部定义 `sections` 数组：

```typescript
export default function PersonalWebsiteProjectPage() {
  // 👇 这个数组决定了左侧目录显示什么
  const sections = [
    { id: "overview", title: "项目概述" },
    { id: "background", title: "项目背景" },
    { id: "architecture", title: "技术架构" },
    { id: "features", title: "实现步骤" },
    { id: "implementation", title: "项目成果" },
    { id: "results", title: "优化展望" },
  ]

  return (
    <ProjectDetailLayout
      sections={sections}  // 👈 传递给布局组件
      // ... 其他属性
    >
      {/* 章节内容 */}
    </ProjectDetailLayout>
  )
}
```

### 2. 创建对应的章节内容

每个 `sections` 数组中的项目，都需要有一个对应的 `<ProjectSection>` 组件：

```typescript
<ProjectSection id="overview" title="项目概述">
  {/* 章节内容 */}
</ProjectSection>

<ProjectSection id="background" title="项目背景">
  {/* 章节内容 */}
</ProjectSection>

// ... 更多章节
```

**重要：** `ProjectSection` 的 `id` 必须与 `sections` 数组中的 `id` 完全一致！

---

## ✅ 正确示例

```typescript
export default function MyProjectPage() {
  // 步骤1：定义章节列表
  const sections = [
    { id: "intro", title: "项目介绍" },      // id: "intro"
    { id: "tech", title: "技术选型" },       // id: "tech"
    { id: "demo", title: "功能演示" },       // id: "demo"
  ]

  return (
    <ProjectDetailLayout sections={sections} {...otherProps}>
      {/* 步骤2：创建对应的章节 */}
      <ProjectSection id="intro" title="项目介绍">
        <p>这是项目介绍...</p>
      </ProjectSection>

      <ProjectSection id="tech" title="技术选型">
        <p>这是技术选型...</p>
      </ProjectSection>

      <ProjectSection id="demo" title="功能演示">
        <p>这是功能演示...</p>
      </ProjectSection>
    </ProjectDetailLayout>
  )
}
```

**结果：** 左侧目录会显示：
- 项目介绍
- 技术选型
- 功能演示

## ❌ 常见错误

### 错误1：id 不匹配

```typescript
const sections = [
  { id: "overview", title: "项目概述" },  // id 是 "overview"
]

// ❌ 错误：id 不匹配
<ProjectSection id="intro" title="项目概述">
  ...
</ProjectSection>

// ✅ 正确：id 必须一致
<ProjectSection id="overview" title="项目概述">
  ...
</ProjectSection>
```

### 错误2：title 不一致

```typescript
const sections = [
  { id: "overview", title: "项目概述" },  // title 是 "项目概述"
]

// ⚠️ 不推荐：title 不一致会造成混淆
<ProjectSection id="overview" title="项目介绍">
  ...
</ProjectSection>

// ✅ 推荐：title 保持一致
<ProjectSection id="overview" title="项目概述">
  ...
</ProjectSection>
```

### 错误3：缺少章节

```typescript
const sections = [
  { id: "overview", title: "项目概述" },
  { id: "tech", title: "技术架构" },
]

// ❌ 错误：只定义了 overview，缺少 tech
<ProjectSection id="overview" title="项目概述">
  ...
</ProjectSection>
// 缺少 id="tech" 的章节！
```

## 🎨 目录功能特性

### 1. 自动高亮当前章节
当你滚动页面时，目录会自动高亮当前正在查看的章节。

### 2. 点击跳转
点击目录中的任意章节，页面会平滑滚动到对应位置。

### 3. 隐藏式显示
- 桌面端：鼠标移到屏幕左侧，目录自动滑出
- 移动端：点击菜单按钮展开目录

### 4. 完全动态
每个项目可以有不同的章节结构，互不影响。

## 📝 如何修改章节

### 添加新章节

```typescript
// 步骤1：在 sections 数组中添加
const sections = [
  { id: "overview", title: "项目概述" },
  { id: "background", title: "项目背景" },
  { id: "newSection", title: "新章节" },  // 👈 添加这一行
]

// 步骤2：添加对应的 ProjectSection
<ProjectSection id="newSection" title="新章节">
  <p>这是新章节的内容...</p>
</ProjectSection>
```

### 删除章节

```typescript
// 步骤1：从 sections 数组中删除
const sections = [
  { id: "overview", title: "项目概述" },
  // { id: "background", title: "项目背景" },  // 👈 注释或删除
]

// 步骤2：删除对应的 ProjectSection
// <ProjectSection id="background" title="项目背景">
//   ...
// </ProjectSection>
```

### 修改章节标题

```typescript
// 步骤1：修改 sections 数组中的 title
const sections = [
  { id: "overview", title: "项目简介" },  // 👈 从"项目概述"改为"项目简介"
]

// 步骤2：修改 ProjectSection 的 title（可选，但推荐保持一致）
<ProjectSection id="overview" title="项目简介">
  ...
</ProjectSection>
```

### 调整章节顺序

```typescript
// 只需调整 sections 数组的顺序，目录会自动更新
const sections = [
  { id: "background", title: "项目背景" },    // 👈 移到第一位
  { id: "overview", title: "项目概述" },      // 👈 移到第二位
  { id: "features", title: "核心功能" },
]

// ProjectSection 的顺序也要对应调整
<ProjectSection id="background" title="项目背景">...</ProjectSection>
<ProjectSection id="overview" title="项目概述">...</ProjectSection>
<ProjectSection id="features" title="核心功能">...</ProjectSection>
```

## 💡 最佳实践

### 1. 章节数量建议
- 最少：3个章节
- 最多：8个章节
- 推荐：4-6个章节

### 2. 章节命名建议
使用简洁、清晰的标题：
- ✅ "项目概述"、"技术架构"、"核心功能"
- ❌ "关于这个项目的一些介绍"（太长）

### 3. id 命名建议
使用英文小写，单词间用连字符：
- ✅ `id: "overview"`, `id: "tech-stack"`, `id: "core-features"`
- ❌ `id: "项目概述"`, `id: "TechStack"`, `id: "core_features"`

### 4. 保持一致性
在同一个项目中，建议使用统一的章节结构风格。

## 🎯 快速检查清单

创建新项目页面时，请确认：

- [ ] 已定义 `sections` 数组
- [ ] 每个 section 都有 `id` 和 `title`
- [ ] 每个 section 都有对应的 `<ProjectSection>`
- [ ] `ProjectSection` 的 `id` 与 `sections` 数组中的 `id` 一致
- [ ] `ProjectSection` 的 `title` 与 `sections` 数组中的 `title` 一致
- [ ] `sections` 数组已传递给 `<ProjectDetailLayout>`
- [ ] 章节顺序合理

## 🆘 常见问题

**Q: 为什么目录没有显示？**
A: 检查是否将 `sections` 数组传递给了 `<ProjectDetailLayout sections={sections}>`

**Q: 为什么点击目录没有跳转？**
A: 检查 `ProjectSection` 的 `id` 是否与 `sections` 数组中的 `id` 完全一致

**Q: 为什么当前章节没有高亮？**
A: 确保页面已经滚动到对应章节的位置，系统会自动检测并高亮

**Q: 可以动态添加章节吗？**
A: 可以！只需修改 `sections` 数组和添加对应的 `<ProjectSection>`，保存后刷新页面即可

---

## 📚 总结

目录系统是**完全动态**的：
1. 在 `sections` 数组中定义章节
2. 创建对应的 `<ProjectSection>` 组件
3. 确保 `id` 匹配
4. 目录自动生成、自动高亮、自动跳转

每个项目都可以有自己独特的章节结构，互不影响！
